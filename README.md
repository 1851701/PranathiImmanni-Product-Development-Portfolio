## Who I am

I'm **Pranathi Immanni**, a **product designer** working at the intersection of **human-centered design**, **cognitive science**, and **digital product development**. I care about how people **think**, **feel**, and **move** through interfaces — especially when stakes are high (health, emotion, trust) or when users are tired, rushed, or overwhelmed.

**What drives my work**

- **Clarity over noise** — hierarchy, typography, and motion that guide attention instead of competing for it.
- **Evidence over assumption** — research-informed flows, ethical framing around AI, and language that matches what the UI actually does.
- **Calm systems** — experiences that feel generous with patience: soft affordances, accessible defaults, and feedback that helps people recover from errors without shame.

I believe **good design should feel calm and obvious**: reducing cognitive load so people can focus on what matters, not on decoding the interface.

## Portfolio highlights *(this repository)*

This repo is my **Next.js** portfolio site: case studies, resume, contact, motion, and theming. Below is what each featured piece explores — for full narrative and screens, clone the repo and run locally, or deploy your own fork.

### Teddy — mental wellness UI

**Calm mobile concept** for mood check-ins, journaling, and gentle insights — designed so reflection feels like a pause, not a performance. Explores **emotional safety**: warm surfaces, rounded components, calm typography, and copy that does not rush the user. Includes live UI in the browser for portfolio presentation.

**Tags:** Mobile · Wellness · UI · Figma · React · Tailwind

---

### CareBridge — healthcare + AI triage *(concept)*

**Web patient portal** pairing an **AI-assisted symptom checker** with labs, appointments, and messaging. Emphasizes **trust**, **accessibility**, and **transparent AI limits** — the kind of clarity expected in real clinical-adjacent products.

**Tags:** Healthcare UX · Web · AI · Accessibility · IA

---

### SkinWise — AI + privacy *(in progress)*

**AI-assisted mobile experience** for skin-condition insights from imagery, with **privacy-first** architecture: Flask APIs, encrypted SQL storage, authentication on sensitive paths, and responsible handling of health-adjacent data. Bridges **data science discipline** with **calm, trustworthy** interface design.

**Tags:** AI · Mobile · Python · Flask · Computer vision · Encryption · Auth

---

### UI Game Portfolio

**Web experience** with game influenced art direction and a tight interaction loop. The case study pairs full width captures of the live site with an embed so visitors can see tone and try the loop. **Pranathi Immanni** — product designer; **Kushal Kongara** — software engineer ([portfolio](https://kushalkongara.vercel.app/)).

**Live demo:** [ux-game-portfolio.vercel.app](https://ux-game-portfolio.vercel.app/)

<details>
<summary><strong>Developer setup — clone & run this portfolio</strong></summary>

```bash
git clone https://github.com/1851701/PranathiImmanni-Product-Development-Portfolio.git
cd PranathiImmanni-Product-Development-Portfolio
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

**Production**

```bash
npm run build
npm start
```

**Where to edit content**

| What | Where |
|------|--------|
| Name, bio, hero, socials, education line | `src/lib/site.ts` |
| Project list & case study metadata | `src/data/projects.ts` |
| Long-form case study copy | `src/data/*-case-study.ts` |
| Resume page body | `src/components/resume/resume-section.tsx` |
| Resume PDF download | add `public/resume.pdf` |
| SEO / OG / sitemap base URL | `.env.local` → `NEXT_PUBLIC_SITE_URL` (see `.env.example`) |

**Stack notes:** App Router, Tailwind v4, Framer Motion, next-themes, contact API route (`src/app/api/contact/route`), `sitemap.ts` / `robots.ts`. Placeholder art may use [picsum.photos](https://picsum.photos/) — replace in `projects.ts` or under `public/` per `next.config.ts` `images` config.
